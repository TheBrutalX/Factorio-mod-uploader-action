# .factorioignore

The `.factorioignore` file allows you to specify patterns for files and directories that should be ignored during the mod upload process. This helps in excluding unnecessary files such as temporary files, logs, and development directories.

## Default Patterns

The following patterns are always applied, even if not included in the `.factorioignore` file:

- `.factorioignore`
- `.git`
- `.github`
- `.gitignore`
- `.vscode`
- `*.tmp`
- `*.log`
- `*.bak`

## Limitations

1. The `**` pattern alone is not supported.
2. Only the following characters are allowed in patterns:
    - Letters (a-z, A-Z)
    - Numbers (0-9)
    - Wildcards (`*`, `?`)
    - Path separators (`/`, `\`)
    - Other special characters: `.`, `_`, `-`

## Enhancing Possibilities with .factorioignore

By using the `.factorioignore` file, you can:

- Exclude development and configuration files from the upload.
- Keep your mod package clean and lightweight.
- Ensure sensitive or irrelevant files are not uploaded.

## Pattern Syntax

### Supported Pattern Types

- **Basic patterns**: `filename.txt`, `directory/`
- **Wildcards**: `*.txt`, `build/*`
- **Directory-specific**: `src/tests/`
- **Root-relative patterns**: `/readme.txt`
- **Negation patterns**: `!important.txt`
- **Multi-directory wildcards**: `**/temp`, `src/**/test`

### Pattern Examples

```gitignore
# Ignore all text files
*.txt

# Ignore specific file in root
/config.json

# Ignore specific directory
/build/

# Ignore temp directories anywhere
**/temp/

# Keep specific files even if in ignored directory
!build/important.txt

# Ignore files in any test directory
**/tests/**
```

## Best Practices

1. Use forward slashes (`/`) for paths, even on Windows.
2. Start patterns with `/` to match from the root directory.
3. Use negation patterns sparingly to maintain clarity.
4. Place more specific patterns before general ones.
5. Comment your patterns with `#` for better maintainability.

## Common Issues

1. **Case Sensitivity**: All patterns are case-insensitive.
2. **Path Separators**: Both forward and backslashes are converted to forward slashes.
3. **Invalid Characters**: Unsupported characters are automatically removed.
